JavaScript Form Handling

As web users, we all know web forms are inherintely awful. We have been there far too many times to count. We fill out a incredibly long form, we finally press the submit button, only to find out we didn't satisfy one of the validation requirements. Even worse, with most server-side form validation, when you get redirected back, you realize you must fill out the entire form again.

I often wondered, why are web forms so bad? How could they be improved? I decided I could do better. My goal was to create the "less painful" web form, with a "as-good-as-possible" user experience. I knew I wouldn't be able to make a form fun, but at least I could prevent people from pulling their hair out.

My form uses JavaScript to impliment client-side form validation, and presents the user with real-time form validation. No more guessing. This also allowed me to prompt the user with descriptive validation requirements, or (form hints). I also created a local storage implementation to save users' non-sensitive data. This prevents the user from having to re-fill out the form if the browser crashes, or they accidentually close the page.

Harvard University Course Catalog using XML data

My final project for my 'Web Development Using XML' course at Harvard Extension School was to take raw XML course data, and develop a robust course catalog website. Users (students) have the ability to search by department, professor, and keyword. A option to search by schedule would display all available courses at a specified semester and time. An advanced search feature allowed students to use multiple different search criteria at once.

College course final project, Harvard University Extension School, Cambridge, MA
